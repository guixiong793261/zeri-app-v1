当前`八字工具整合.py`中的函数命名存在不一致：
- 部分函数使用`ba_`前缀（如`ba_get_shishen`）
- 部分函数使用中文命名（如`get_shishen`）
- 部分函数使用英文命名（如`calculate_wuxing_score`）

## 二、整合方案

### 2.1 命名规范

建议采用**中文命名为主，英文命名为辅**的策略：

#### 基础数据获取函数
```python
# 使用 get_ 前缀 + 中文描述
def get_gan_wuxing(gan: str) -> str: ...
def get_zhi_wuxing(zhi: str) -> str: ...
def get_gan_yinyang(gan: str) -> str: ...
def get_zhi_yinyang(zhi: str) -> str: ...
```

#### 计算函数
```python
# 使用 calculate_ 前缀 + 中文描述
def calculate_shishen(day_gan: str, target_gan: str) -> str: ...
def calculate_wuxing_score(sizhu: Dict, ...) -> Dict[str, float]: ...
def calculate_zhangsheng(day_gan: str, zhi: str) -> str: ...
```

#### 检查函数
```python
# 使用 check_ 前缀 + 中文描述
def check_liuhe(zhi1: str, zhi2: str) -> Optional[str]: ...
def check_liuchong(zhi1: str, zhi2: str) -> bool: ...
def check_sanhe(zhis: List[str]) -> Optional[str]: ...
```

### 2.2 模块整合策略

#### 方案A：保留独立文件（推荐）

保留`八字工具整合.py`作为独立模块，原因：
1. 功能较多且相对独立
2. 便于维护和更新
3. 避免`工具函数.py`过于臃肿
4. 清晰的模块职责划分

#### 方案B：合并到工具函数.py

将`八字工具整合.py`合并到`工具函数.py`：
- 优点：减少文件数量
- 缺点：文件过大，职责不清晰
- 建议：不采用

## 六、使用指南

### 6.1 新模块开发

```python

    GAN_WUXING, ZHI_WUXING,
```

### 6.2 旧模块迁移

逐步将旧模块中的函数调用迁移到`八字工具整合`：

```python

# 新代码
from .八字工具整合 import get_gan_wuxing
```

### 6.3 保持向后兼容

如果必须使用`工具函数.py`，可以添加重定向：

```python

    get_gan_wuxing,
    get_zhi_wuxing,
    # ... 其他函数
)
```
