# 喜用神计算器优化总结

## 一、优化目标

统一使用八字工具整合模块的加权算法，提高喜用神计算的精度。

## 二、优化内容

### 2.1 算法对比

| 项目 | 旧算法 | 新算法 | 优势 |
|------|--------|--------|------|
| 天干五行 | 统计次数 | 权重1.0 | 更精确 |
| 地支本气 | 统计次数 | 权重0.8 | 考虑地支本气 |
| 藏干 | 统计次数 | 主气0.6×0.5，中气0.3×0.5，余气0.1×0.5 | 考虑藏干权重 |
| 阈值 | 整数（>4, >2, <2） | 浮点数（>6.0, >3.0, <2.0） | 更精细 |

### 2.2 具体改进

#### 改进1：使用加权算法

**旧代码**（仅统计次数）：
```python
# 统计天干五行（每次+1）
for gan_key in ['year_gan', 'month_gan', 'day_gan', 'hour_gan']:
    gan = sizhu.get(gan_key, '')
    if gan:
        wx = GAN_WUXING.get(gan, '')
        if wx:
            wuxing_counts[wx] += 1

# 统计地支藏干（每次+1）
for zhi in ZHIGAN_MAP[zhi]:
    wuxing_counts[zhi] += 1
```

**新代码**（使用加权算法）：
```python
# 使用整合模块的加权算法
try:
    return calculate_wuxing_score(sizhu, include_canggan=True, use_weight=True)
except Exception as e:
    logger.warning(f"使用加权算法失败，回退到简化算法: {e}")
    # 回退到简化算法
```

**加权算法详情**：
- 天干五行：权重1.0
- 地支本气：权重0.8
- 藏干主气：权重0.6 × 0.5 = 0.3
- 藏干中气：权重0.3 × 0.5 = 0.15
- 藏干余气：权重0.1 × 0.5 = 0.05

#### 改进2：调整阈值

**旧阈值**（整数）：
```python
if gold_count > 4:      # 太旺
elif fire_count > 2:     # 旺
elif earth_count < 2:     # 弱
```

**新阈值**（浮点数）：
```python
if gold_count > 6.0:     # 太旺
elif fire_count > 3.0:    # 旺
elif earth_count < 2.0:    # 弱
```

**阈值调整原因**：
- 加权算法的分数范围更大（0-10+）
- 需要相应提高阈值以保持判断准确性
- 使用浮点数可以更精细地区分五行强弱
