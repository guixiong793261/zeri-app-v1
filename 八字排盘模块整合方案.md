# 八字排盘模块整合方案

## 一、模块评估

### 1.1 功能概述

| 功能 | 说明 | 状态 |
|------|------|------|
| 四柱计算 | 依赖高精度农历转换，有备用简化算法 | ✅ 与现有项目一致 |
| 藏干 | 基于预定义的ZHIGAN_MAP提取地支藏干 | ⚠️ 需要使用带权重版本 |
| 十神 | 基于阴阳和五生克的严谨算法 | ✅ 优于旧版，应替换 |
| 五行统计 | 统计天干地支的五行个数（不含权重） | ⚠️ 需要补充权重计算 |
| 纳音 | 查询六十甲子纳音，年、日柱完整 | ✅ 可保留 |
| 十二长生 | 计算日主在地支的十二长生状态 | ✅ 可保留 |
| 大运 | 根据性别和年干阴阳排大运 | ✅ 可保留，需完善起运年龄 |

### 1.2 与现有项目的关系

| 项目 | 八字排盘模块 | 现有项目 | 整合建议 |
|------|-------------|----------|----------|
| 基础数据 | 本地定义TIAN_GAN、DI_ZHI等 | 工具函数.py中定义 | 从工具函数.py导入 |
| 十神函数 | 基于阴阳和生克的算法 | 基于映射表的旧版 | 替换为新版算法 |
| 藏干表 | 简化版（仅天干列表） | 带权重版本 | 使用带权重版本 |
| 四柱计算 | 返回格式：'year'、'year_gan'等 | 部分返回：'年柱'、'日柱'等 | 统一格式或添加兼容层 |
| 五行统计 | 简单计数 | 带权重的精细计算 | 补充权重计算 |
| 纳音 | 年、日柱完整 | 无 | 保留并完善 |
| 十二长生 | 完整实现 | 无 | 保留 |
| 大运 | 基础实现 | 无 | 保留并完善 |

### 2.3 藏干表整合

**修改方案**：
- 使用工具函数.py中的ZHIGAN_WEIGHTED（带权重）
- 在藏干显示时仅提取天干列表，权重用于内部计算

**代码示例**：
```python
def _calculate_canggan(self):
    """计算藏干"""
    self.canggan = {}
    for zhi_name, zhi_key in [('年支', 'year_zhi'), ('月支', 'month_zhi'), 
                            ('日支', 'day_zhi'), ('时支', 'hour_zhi')]:
        zhi = self.sizhu.get(zhi_key, '')
        if zhi in ZHIGAN_MAP:
            # 提取天干列表（忽略权重）
            if isinstance(ZHIGAN_MAP[zhi], list):
                # 简化版：直接使用列表
                self.canggan[zhi_name] = ZHIGAN_MAP[zhi]
            else:
                # 带权重版：提取天干
                self.canggan[zhi_name] = [gan for gan, _ in ZHIGAN_MAP[zhi]]
        else:
            self.canggan[zhi_name] = []
```

    """计算五行统计"""
    # 基础计数

    # 天干五行

## 五、风险评估

### 5.1 潜在风险

1. **数据格式不一致**：不同模块返回的四柱格式可能不一致
2. **算法差异**：新旧十神算法可能产生不同结果
3. **性能影响**：新增功能可能影响排盘速度
4. **依赖冲突**：模块间的依赖关系可能产生冲突

### 5.2 风险对策

1. **统一数据格式**：添加格式转换层，确保数据格式一致
2. **算法验证**：对比新旧算法结果，确保准确性
3. **性能优化**：优化计算逻辑，减少不必要的计算
4. **依赖管理**：明确模块间的依赖关系，避免循环依赖
