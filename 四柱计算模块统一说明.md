# 四柱计算模块统一说明

## 概述
已将项目中的四柱计算模块统一为使用 `HighPrecisionLunar` 作为唯一底层计算引擎，支持双库校验（sxtwl + lunar-python），确保计算精度和准确性。

## 主要改进

### 1. 统一计算引擎
- **之前**：`四柱计算器.py` 混合使用 lunar_python 和 sxtwl，`高精度四柱计算器.py` 仅使用 lunar_python
- **现在**：统一使用 `高精度农历转换.py` 中的 `HighPrecisionLunar` 类作为唯一入口

### 2. 双库校验机制
- **主库**：sxtwl（天文算法，范围广，精度高）
- **校验库**：lunar-python（数据表查询，功能全面）
- **优势**：在节气交界日自动检测差异，确保计算准确性

### 3. 保持接口兼容
所有现有接口保持不变，确保向后兼容：
- `calculate_sizhu(target_date, hour=12, minute=0, second=0)` - 计算四柱
- `analyze_sizhu(sizhu)` - 分析四柱（五行、十神、夫星子星）
- `get_lunar_date(target_date, hour=12, minute=0, second=0)` - 获取农历日期
- `validate_sizhu(target_date, hour=12, minute=0)` - 验证四柱准确性

### 4. 新增功能
- **strict_mode**：严格模式，校验不一致时抛出异常（适合调试）
- **双库校验**：自动比较 sxtwl 和 lunar-python 的结果
- **校验标记**：返回结果中包含 `verified` 标记，表示是否通过双库校验

## 使用示例

### 基本使用
```python
from modules.四柱计算器 import calculate_sizhu, analyze_sizhu
from datetime import date

# 计算四柱
d = date(1972, 1, 8)
sizhu = calculate_sizhu(d, hour=8, minute=0)

# 输出结果
print(f"年柱: {sizhu['年柱']}")
print(f"月柱: {sizhu['月柱']}")
print(f"日柱: {sizhu['日柱']}")
print(f"时柱: {sizhu['时柱']}")

# 分析四柱
analysis = analyze_sizhu(sizhu)
print(f"五行: {analysis['五行']}")
print(f"十神: {analysis['十神']}")
```

### 严格模式
```python
from modules.四柱计算器 import SiZhuCalculator

# 创建严格模式的计算器
calculator = SiZhuCalculator(strict_mode=True)

# 计算（校验不一致时会抛出异常）
sizhu = calculator.calculate(date(1972, 1, 8), hour=8)
```

### 验证四柱
```python
from modules.四柱计算器 import validate_sizhu

# 验证四柱计算的准确性
is_valid, message, diffs = validate_sizhu(date(1972, 1, 8), hour=8)

if is_valid:
    print(f"✅ {message}")
else:
    print(f"❌ {message}")
    if diffs:
        print("差异详情:")
        for diff in diffs:
            print(f"  - {diff}")
```

## 优势总结

1. **高精度**：使用 sxtwl 天文算法，计算精度更高
2. **双库校验**：自动检测计算差异，确保准确性
3. **统一接口**：所有模块使用相同的计算引擎，结果一致
4. **向后兼容**：保持现有接口不变，无需修改调用代码
5. **易于维护**：单一计算引擎，便于维护和升级
6. **节气边界**：精确处理节气交界日，避免月柱计算错误

## 注意事项

1. **节气交界日**：在节气交界日（如立春、惊蛰等）前后几天，sxtwl 和 lunar-python 的月柱可能不一致，这是正常现象
2. **建议**：
   - 普通择日：使用默认模式（strict_mode=False），以 sxtwl 结果为准
   - 精确择日：在节气交界日前后几天，建议人工复核
   - 严格模式：strict_mode=True 会在不一致时抛出异常，适合调试使用
3. **影响范围**：年柱、日柱、时柱通常一致，主要影响月柱相关的分析
